# サカイオートサービス｜顧客管理・予約・車検リマインド デモ

自動車整備工場向けの営業提案用デモアプリ。紙台帳/Excel運用の中小整備工場に「実際に動く」サンプルとして見せることを目的にした、単一の静的Webアプリです。

## 特徴（実際に動きます・見た目だけのモックではありません）

- **2ビュー切替**（画面上部タブ）
  - 🖥 **店舗側（管理画面）**: 計器/ダッシュボード風のダークUI
  - 📱 **お客様側（スマホ）**: スマホモックフレーム内にLINEトーク風UI
- **店舗側の機能**
  - 顧客・車両一覧（顧客名/車種/ナンバー/車検満了日/前回来店日/次のアクション）
  - 車検満了日から「期限切れ / 1ヶ月以内 / 3ヶ月以内 / 余裕あり」を色分けバッジで自動判定
  - 顧客・車両の新規登録／編集
  - 予約管理（日付ごと一覧、同一時間帯の重複バッティングを警告表示）
  - 🔔 車検リマインド通知の**一括自動生成**（対象顧客を自動抽出）
  - 手動通知の発行（修理完了 / タイヤ交換の季節案内 など。種類選択で定型文プリセット）
  - 通知一覧（各通知に「実運用ではLINEに自動送信されます」の補足）
- **お客様側の機能**
  - その顧客宛の通知をLINEトーク風に時系列表示（車検リマインド/予約確認/修理完了/タイヤ交換）
  - 各通知に「※実際の運用ではLINEに届きます」の補足
  - 予約リクエストフォーム → 送信すると**店舗側の予約・通知に即反映**（両ビューがデータで連動）
- **データ永続化**: ブラウザの localStorage。リロードしても消えません。
- **デモデータ**: 架空の「サカイオートサービス」(大阪府堺市)＋日本人顧客10件。車検満了日は開いた日を基準に動的生成され、常に4ステータスが混在します。

## 技術構成

- 静的SPA（ビルド不要 / 依存ライブラリなし / Vanilla JS）
- 保存先: localStorage（バックエンド不要 = 最もシンプル）
- ファイル
  - `index.html` … 骨格・スクリプト読込
  - `styles.css` … 計器風ダーク＋琥珀テーマ / スマホモック / レスポンシブ
  - `data.js` … 状態モデル・永続化(`Store`)・日付/車検判定(`Utils`)・デモデータseed
  - `notifications.js` … 通知の**生成**(`Notifications`)と**送信の出口**(`Channel`)を分離
  - `admin.js` … 店舗側ビュー(`AdminView`)
  - `customer.js` … お客様側ビュー(`CustomerView`)
  - `app.js` … 統合・ビュー切替(`App`)

### 将来のLINE連携ポイント

通知の「生成」と「送信」を意図的に分離しています。本番でLINE公式アカウントへ配信するには、
`notifications.js` の `Channel.deliver()` 内のコメント箇所に LINE Messaging API の push message 送信を差し込むだけです（顧客に `lineUserId` を持たせる）。生成側（`Notifications.generate*` / `issue`）は変更不要です。

## ローカルで開く

`index.html` をブラウザで開くだけで動きます（サーバー不要）。

## Netlify へのデプロイ

- **ドラッグ&ドロップ**: Netlifyの管理画面に `sakai-auto-demo` フォルダをドロップ。
- **Git連携**: リポジトリを繋ぐと `netlify.toml`（publish = "."）でそのまま公開されます。ビルドコマンドは不要です。

## データのリセット

ブラウザの localStorage キー `sakai_auto_state_v1` を削除（開発者ツール → Application → Local Storage）すると、初期デモデータで再生成されます。
